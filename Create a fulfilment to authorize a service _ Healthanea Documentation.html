Create a fulfilment to authorize a service

See also the API specification (Swagger) and Error management

See also:

  • Concept: Service fulfilment
  • API documentation: The Fulfilment API

Producer Channel role: You use the Fulfilment API to create a fulfilment instance with which you authorize Service Providers to offer healthcare services to end-users.

Service role: You use the Fulfilment API to register actions related to your service's lifecycle within the context of the fulfilment instance. These can include actions performed by the end-user but also the read and write actions applied to the end-user's health data.

The Fulfilment API tracks the actions performed by the health services and the end-user, thus recording a customer health journey.

Fulfilment API flow

The following diagram illustrates the flow of Fulfilment API usage:


1 - Create a fulfilment instance

Role: Producer Channel
Healthanea API: Fulfilment
Mandatory/optional: Mandatory

If you do not have one already, get a token for the Producer Channel role (see Authentication & tokens).

Use a POST on the fulfilment/instance endpoint to create a fulfilment instance:

POST https://api-nxt.dhpbuild.com/api/dhp/v1/fulfilment/instance

You must include a request body that contains:

  • A journeyId: A session identifier, managed by the Producer Channel, used to track one or more service executions for a single end-user during a single healthcare journey. (The maximum allowed length of the journeyId is 100 characters.)
  • A dhpUserId: The identifier of the end-user on the Healthanea platform.
  • A serviceId: The identifier of a specific health service to authorize to interact with the Healthanea platform.
  • (Optional) A deviceId: The identifier of the end-user's device associated with the health journey. (The maximum allowed length of the deviceId is 100 characters.)
  • (Optional) A startDate: The start date and time of the fulfilment. This is used for asynchronous sending of fulfilment data to the Healthanea platform. See Sending service fulfilment data asynchronously.

An example of a request body is the following:

{
  "journeyId": "{{journeyId}}",
  "dhpUserId": "{{dhpUserId}}",
  "serviceId": "{{serviceId}}",
  "deviceId": "{{deviceId}}",
  "startDate": "{{startDate}}"
}

If the request is successful, Healthanea returns a 201 HTTP status code and a response body containing the new fulfilment ID:

{
   "fulfilmentId": "{{fulfilmentId}}"
}

2 - Add an action to the fulfilment instance

Roles: Producer Channel & Service
Healthanea API: Fulfilment
Mandatory/optional: Optional

If you do not have one already, get a token for the Producer Channel role (see Authentication & tokens).

Use a POST on the fulfilment/instance/{fulfilmentId}/actions endpoint to add an action to the instance for the Healthanea platform to track:

POST https://api-nxt.dhpbuild.com/api/dhp/v1/fulfilment/instance/{fulfilmentId}/actions

In the request URL, you must include a fulfilmentId, the identifier of the fulfilment instance to which to add the action.

You must also include a request body that contains:

  • An actionTypeId: The identifier of the action type. The action type ID indicates the action "template"; when one is added to the fulfilment instance, a specific instance of the action is recorded.
  • (Optional) A payload: Any data associated with the action, structured as a valid JSON object.
  • (Optional) An actionDate: The date of the action; this is used for asynchronous sending of fulfilment data to the Healthanea platform. See Sending service fulfilment data asynchronously.

Although actions are not mandatory, they give you a detailed view of how fulfilments progress. The more actions you push, the more you can record what the end-user is doing.

An example of a request body is the following:

{
  "actionTypeId": "{{actionTypeId}}",
  "payload":
  {
    "key0": "string-value",
    "key1": 12
  },
  "actionDate": "{{actionDate}}"
}

If the request is successful, Healthanea returns a 204 HTTP status code to confirm that the action was added to the instance.

3 - Add context properties to the fulfilment instance

Roles: Producer Channel & Service
Healthanea API: Fulfilment
Mandatory/optional: Optional

If you do not have one already, get a token for the Producer Channel role (see Authentication & tokens).

Use a POST on the fulfilment/instance/{fulfilmentId}/context endpoint to add further information to the fulfilment instance to provide additional context:

POST https://api-nxt.dhpbuild.com/api/dhp/v1/fulfilment/instance/{fulfilmentId}/context

In the request URL, you must include a fulfilmentId, the identifier of the fulfilment instance to which to add the context properties.

You must also include a request body that contains a properties object containing one or more key-value pairs of name and value. An example of a request body is the following:

{
  "properties": [
    {
      "name": "{{name1}}",
      "value": "{{value}}"
    },
    {
      "name": "{{name2}}",
      "value": "{{value}}"
    }
  ]
}

If the request is successful, Healthanea returns a 202 HTTP status code confirming that the context information was added to the fulfilment instance.

4 - Get the properties of the fulfilment instance

Roles: Producer Channel & Service
Healthanea API: Fulfilment
Mandatory/optional: Optional

If you do not have one already, get a token for the Producer Channel role (see Authentication & tokens).

Use a GET on the fulfilment/instance/{fulfilmentId} endpoint to read the properties of an existing fulfilment instance:

GET https://api-nxt.dhpbuild.com/api/dhp/v1/fulfilment/instance/{fulfilmentId}

In the request URL, you must include a fulfilmentId, the identifier of the fulfilment instance whose properties you want to retrieve.

If the request is successful, Healthanea returns a 200 HTTP status code with a response payload containing the properties of the fulfilment instance in the following format:

{
  "id": "{{id}}",
  "producerId": "{{producerId}}",
  "producerChannelId": "{{producerChannelId}}",
  "serviceId": "{{serviceId}}",
  "serviceProviderId": "{{serviceProviderId}}",
  "journeyId": "{{journeyId}}",
  "deviceId": "{{deviceId}}",
  "dhpUserId": "{{dhpUserId}}",
  "timestamp": "{{timestamp}}",
  "expirationTimestamp": "{{expirationTimestamp}}",
  "dataExchangeCompletionActionTypeIds": [
    "{{dataExchangeCompletionActionTypeId}}"
  ]
}

5 - Get the payload of an action in a fulfilment instance

Roles: Producer Channel & Service
Healthanea API: Fulfilment
Mandatory/optional: Optional

If you do not have one already, get a token for the Producer Channel role (see Authentication & tokens).

Use a GET on the fulfilment/instance/{fulfilmentId}/actions endpoint to get the payload of a specific action in a fulfilment instance:

GET https://api-nxt.dhpbuild.com/api/dhp/v1/fulfilment/instance/{fulfilmentId}/actions/{actionTypeId}

In the request URL, you must include:

  • A fulfilmentId: The identifier of the fulfilment instance from which you want to get the action payload.
  • An actionTypeId: The identifier of the action type.

If the request is successful, Healthanea returns a 200 HTTP status code and a response body containing the details of the action payload for the last action in the fulfilment having the specified action type ID:

{
  "actionId": "{{actionId}}",
  "actionTypeId": "{{actionTypeId}}",
  "timeStamp": "{{timeStamp}}",
  "payload": {}
}

Viewing requests in the API explorer

In the Healthanea portal's API explorer, you can review all the API requests that you executed above. This is useful to check that the operations completed successfully.
