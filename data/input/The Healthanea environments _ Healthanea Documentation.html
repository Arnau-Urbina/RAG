The Healthanea environments

Roles: All

Healthanea provides an API and a portal for administration, analytics, and documentation. All these tools are available on multiple environments. This article describes the purpose of the different environments, how each should be used, and how to access them.

In this article:

  • The environments
  • The Healthanea API
  • API troubleshooting per environment
  • The Healthanea portal: administration, analytics, & documentation

The environments

The following table describes the environments available on the Healthanea platform:

Name Data Usage URL
Production Live data For live production usage by real end-users.
This environment must not be used for testing purposes. 		https://i01.dhp-services.io/api/dhp/
Test & integration (NXT) Test and Healthanea partner development data For Healthanea partners to:
  • develop a new integration of Healthanea
  • validate a new Healthanea version prior to its release on Production
  • perform user acceptance testing

    The Healthanea API

    The Healthanea API is exposed independently on each environment. Note that some features available on NXT may not yet be available on Production.

    Each environment requires dedicated API credentials. See Authentication & tokens.

    API troubleshooting per environment

    A request returns a 401 status code

    An API request that returns the 401 status code “unauthorized” indicates that you provided an expired or invalid token for the specific environment. See Authentication & tokens.

    A request returns a 404 status code

    An API request that returns the 404 status code “not found” indicates that a given resource ID was not found. Although resource IDs exist in all environments, some IDs are specific to a given environment. Check that the resource ID you provided exists in the given environment.

    The Healthanea portal: administration, analytics, & documentation

    The Healthanea portal allows access to administrative tools, analytics, and the Healthanea product documentation. Like the API, the portal is deployed independently on each environment. The portal provides access only to the data associated with the specific environment.

    The URLs for the Healthanea portal per environment are the following:

    Environment URL
    Production https://i01.dhp-services.io/web/portal/
    Test & integration (NXT) https://api-nxt.dhpbuild.com/web/portal/

    Note that some features available on the Healthanea portal on NXT may not yet be available on Production.
