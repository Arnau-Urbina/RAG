The Mediktor widget

Healthanea is an API-based platform that does not provide user interfaces. This decision was made at the beginning of the Healthanea project after acknowledging that every Producer may have multiple channels that will always be different. Each Producer Channel will need to be tailored to a specific country, user target, type of business, and so on.

Therefore, to integrate with Healthanea means to integrate with its APIs. The most fundamental APIs are the Universal ID API that creates a DHP user ID and the Fulfilment API that authorizes a service and tracks usage events.

Nonetheless, as Mediktor is a pre-integrated service which already offers a web UI SDK, the Healthanea team has created a Service Integration Page for Mediktor, more commonly known as the Mediktor widget.

The following diagram shows the differences between the standard approach (option a) and the widget integration pattern (option b):

The main difference is that instead of integrating directly the service UI (Mediktor Web SDK) and orchestrating the different calls to the Healthanea Fulfilment API, a Producer can call the Integration Page API, providing some configuration parameters, obtain a token and either redirect to to the widget URL or integrate it as an iframe. In both cases the widget token must be provided in the URL as a parameter.

Once the integration page is launched, the end-user can interact with the Mediktor UI. The generated events are captured by the widget and tracked in Healthanea. A callback (or message in the case of an iframe integration) is called at the end of the symptom checker session so that the Producer can reclaim control and move to the next step in the customer journey.
