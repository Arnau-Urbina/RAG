What is Healthanea?

Healthanea is a business-driven, secure, compliant, and scalable platform to allow the orchestration of digital healthcare services, with an emphasis on highly secure data exchange. Healthanea is at the forefront of the next generation of digital healthcare use cases that will increase customer satisfaction and enable powerful insights and research by leveraging data.

The Healthanea platform is where health services orchestration and management meet with a powerful personalized health data infrastructure:

The Healthanea portal

The platform control centre is the Healthanea portal, which allows authorized users to:

  • Orchestrate Healthanea assets; for example, manage the collection of services that can be included in a health journey.
  • Monitor KPIs through graphical reports that aggregate end-user and service data.
  • Manage Healthanea user accounts and API credentials.

The Healthanea portal is the place where Producers and Service Providers can access the information and configurations relevant to them.

Population statistics and KPIs

You can monitor and understand how end-users are engaging with health services in the Healthanea portal. Identify trends on service usage, track the main symptoms given to the symptom checker as well as the main diagnoses, discover the most used customer journeys, find out what people tend to do before and after a specific service, and much more.

Population KPIs are always aggregated to respect GDPR requirements.

The Healthanea API

The backbone of the Healthanea platform is a REST API, which enables authenticated users to:

  • Orchestrate healthcare services.
  • Exchange personal health and non-health data.

The Healthanea API consists of a collection of microservices that developers use to orchestrate health services in their Healthanea-powered apps. The main API components are the following:

Fulfilment API

Before a Service Provider can exchange data for a customer of a Producer at a given moment in time, it must get an authorization, called a fulfilment ID, from the Producer. The Fulfilment API is also used to track the usage of the health services, to allow the personalization of services and the generation of insights.

  • Fulfilment concept
  • Fulfilment API

Health API

During the execution of a health service, medical data may be generated. Sometimes previous medical data may be used to personalize the service (for example, symptom checker data as input of a teleconsultation session). The Health API is used to exchange medical data in a secure way. The FHIR standard is used to maximize interoperability between services.

  • Health data exchange concept
  • Health API

Recommendation API

Healthanea's next-best-action feature can be used to generate a list of recommended services personalized for a specific user based on age, gender, country, past history, and so on. Currently, the Recommendation API's proposals are based on decision tables of business rules.

  • Next-best-action concept
  • Recommendation API

Universal ID API

Data privacy is at the heart of all transactions on the Healthanea platform. All the data in Healthanea is either anonymized or pseudonymized. With the Universal ID API, you can easily manage the identity mapping between Healthanea and your local systems.

  • Universal ID API

Authentication with Azure Active Directory

Microsoft Azure Active Directory (AAD) is a powerful and secure identity-as-a-service (IDaaS) system. Healthanea calls AAD to authenticate and get the access tokens required for the authorization headers in all your Healthanea API calls.

Symptom Checker Widget

Do you have a standard symptom checker use case? Do you want to speed up integration and reduce integration and maintenance costs?

The symptom checker widget is an integration page that prepackages a customized instance of the Mediktor SDK. The package includes all the associated Healthanea API calls to create the fulfilment ID and track the events generated by the symptom checker.

The widget can be used in full screen (redirect to the page) or as part of your web page (integrated in an iframe).
